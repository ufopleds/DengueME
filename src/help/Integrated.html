Path

Results Path: Informe o nome da pasta na qual os resultados da simulação serão armazenados. Se você desejar manter os resultados de uma simulação, altere o nome desta pasta antes de executar novamente o modelo.

Attributes to save in results file: Selecione os atributos a serem armazenados como resultados da simulação.

Individual results

Name of results file: Informe o nome do arquivo .csv que armazenará os resultados da simulação.

Cell (Wich will be saved): Cells are identifyed by its x, y position. Use the position of a cell {x, y}. If multiple cells are needed, separate then by comma like {x1, y1}, {x2, y2}.

Attributes to save in results file: Selecione os atributos a serem armazenados como resultados da simulação.

Simulation

Debug mode: Simulação em modo de depuração. Exibe informações detalhadas durante a execução do modelo. Em geral, essa opção é utilizada para testes.

Simulation start/end time: Informe o intervalo de tempo (tempo inicial e tempo final) da simulação.

Save results start/end time: Informe o intervalo de tempo (tempo inicial e tempo final) no qual os resultados serão armazenados.

ODE

Step: Informe o passo de integração das Equações Diferenciais Ordinárias (EDOs) do modelo.

Population Control

Mechanical Control

Rate (0-1): Taxa de retirada de criadouros do ambiente.

Time (When control will be applied): Use {t} for an instant or {t1, t2} for a interval. If multiple times are needed separate then by comma like {t1}, {t2, t3}, {t4}

Cell (Where control will be applied): Cells are identifyed by its x, y position. Use the position of a cell {x, y} or a region {x1, y1, x2, y2}. If multiple spaces are needed, separate then by comma like {x1, y1}, {x2, y2, x3, y3}, {x4, y4}.

Chemical Control

Population Target: População alvo do controle.

Efficacy (0-1): Taxa de mortalidade máxima do inseticida.

Environmental Persistence: Tempo de persistência do inseticida no ambiente.

Time (When control will be applied): Use {t} for an instant or {t1, t2} for a interval. If multiple times are needed separate then by comma like {t1}, {t2, t3}, {t4}

Cell (Where control will be applied): Cells are identifyed by its x, y position. Use the position of a cell {x, y} or a region {x1, y1, x2, y2}. If multiple spaces are needed, separate then by comma like {x1, y1}, {x2, y2, x3, y3}, {x4, y4}.

Constants

Carrying capacity (K):

Informe a capacidade de suporte para o modelo. Essa constante está presente no termo logístico que controla o crescimento da população. Em geral é baseado em características ambientais do local de estudo.

Mortality Rates:

Eggs: Informe a taxa de mortalidade natural da população de ovos.

Larvae: Informe a taxa de mortalidade natural da população de larvas.

Pupae: Informe a taxa de mortalidade natural da população de pupas.

Mosquitoes: Informe a taxa de mortalidade natural da população de adultos.

Development Rates:

Eggs: Informe a taxa de desenvolvimento de ovos.

Larvae: Taxa de desenvolvimento de larvas.

Pupae: Taxa de desenvolvimento de pupas.

Temperature time series:

Informe a temperatura (constante) ou a série temporal de temperatura da região estudada. Se a temperatura for constante ao longo do tempo basta informar o valor desejado. Caso a temperatura seja dada por uma série temporal, é necessário informar os valores das temperaturas separados por vírgula. Neste caso, também é possível importar os valores de temperatura armazenados em um arquivo .csv (sem cabeçalho) através do botão "import from csv". É possível obter estes dados de fontes públicas, por exemplo, INMET e CPTEC.

Population

Population Scale: Informe a escala da população (0 a 1). Este atributo permite escalar a quantidade da população escolhida. Por exemplo, pode-se realizar a simulação utilizando somente 10% da população, definindo o valor como 0.1. Assim, é possível executar o modelo mais rapidamente para a obtenção de resultados e realização de testes.

Population type: Informe o tipo de população que deseja espacializar no modelo. Para isto, basta escolher o tipo de população: oviposition (considera apenas os ovos depositados sem a diminuição desses ovos causada pela morte natural, nesse caso, utilizamos a população de ovos), eggs, larvae, pupae, mosquitoes.

Database

dbType: Informe o tipo de base de dados a ser utilizado pelo modelo. É possível utilizar como base de dados um banco de dados Mysql ou Access. Também é possível criar um espaço celular virtual (grade de células). A base de dados pode ser construída utilizando um Sistema de Informações Geográficas (SIG), como, por exemplo, o TerraView.

Observer

Legend scale: Informe a escala da legenda. Este atributo permite escalar os valores dos atributos de uma legenda. Por exemplo, suponha que o valor da população desejada varia entre 10 e 20 durante a simulação para uma determinada configuração dos parâmetros do modelo. E ao definir uma nova configuração de parâmetros para o modelo, este valor passou a variar entre 100 e 200. Neste caso, não é necessário definir novamente a legenda, basta ajustá-la para a escala adequada - multiplicando-a por 10 neste exemplo.

Type: Defina o tipo do atributo a ser observado. Pode ser: "bool", "number", "string", "datetime" (an ordered string)

Grouping: Informe a estratégia para agrupamento e coloração dos dados. Pode ser um dos seguintes tipos: equalsteps, quantil, stdeviation, uniquevalue.

Slices: Informe o número de cores a ser utilizada na visualização dos resultados.

Precision: Informe a precisão (número de casas decimais) do atributo a ser utilizada para visualização e geração dos resultados.

StdDeviation: Este atributo deve ser informado quando a estratégia de agrupamento escolhida for stdeviation, pondendo assumir um dos seguintes valores: "full", "half" "quarter", "none".

Maximum: Informe o valor máximo do atributo observado.

Mininum: Informe o valor mínimo do atributo observado.

ColorBar: Informe a tabela de cores a ser utilizada na visualização dos resultados. Cada elemento desta tabela é constituído por uma cor e o respectivo valor do atributo. Cada cor pode ser informada como uma string ("red", "green", "blue", "white", "black", "yellow", "brown", "cyan", "gray", "magenta"), ou como uma tabela com três valores representando a sua composição RGB. Por exemplo: "{00, 00, 00}").

Generate map: Selecione este parâmetro para gravar os resultados da simulação como mapas.

View Execution: Selecione este parâmetro se desejar visualizar a execução do modelo.
